# Retrospective — 2026-06-22 Session

## Session scope
Single session covering BnB/Stage bug audit + fix cycle across a SvelteKit + Convex app (ms-ellen-project).

## Bug audit methodology
5-layer decomposition: UI (Svelte) → Stores (state) → Service/Data (sync, IndexedDB) → Convex backend (views/mutations) → Schema (indexes). Each layer explored independently by a subagent, then findings collated into a unified bug list ordered by severity.

## What worked

**Systematic layer audit.** The decomposition surfaced bugs that cross-layer review would miss — auth guard gaps in Convex while UI was being audited, toast race conditions in stores while Convex auth was being hardened. Each layer had clear scope.

**Consistent auth guard pattern.** Standardizing on `requireStaff(ctx, undefined, bypassUserId)` across 10+ query files created a single pattern that's easy to audit going forward. The `bypassUserId` parameter lets PIN auth and public iCal endpoints opt out of the required identity check without duplicating queries.

**SHA gate caught real issues.** The gate flagged two problematic subagent changes: an index name override (`by_cycleId` → `by_cycle`) that didn't exist in the schema (breaking 8 iCal tests), and `as never` → plain string type replacements (breaking 11 type checks). Without the gate these would have been committed and discovered later.

## What didn't work

### Subagent reliability (highest friction)
- `deepseek-v4-flash:free` on `opencode-zen` was consistently broken (401 not supported) — blocked delegation for multiple attempts while config was debugged
- `MiniMax-M2.7` via `minimax.io` works but has 600s timeouts on multi-file changes (2 out of ~8 dispatches timed out at 37 and 28 calls respectively)
- One subagent modified files incorrectly (index name override) — required manual revert + redo
- Another replaced `as never` casts with plain string types, reasoning that "types should match" without understanding that Convex branded `Id<"xxx">` types are intentionally opaque

**Cost:** Each 600s timeout + verify + revert + fix cycle took approximately as long as doing the fix directly. The net benefit of delegation was near zero for multi-file changes.

### Pipeline ceremony overhead
Every task requires 6-7 phase transitions even for 1-line fixes:
```
IMPLEMENT → CONTEXT_GUARD → CRITIQUE → REVIEW → VERIFY → EVAL → RETRO → CONCLUDE
```
Approximately 60% of submission code handles the state machine (RETRY → unblock → resubmit with CONCLUDE PUNCH LIST format) rather than doing actual work.

### CONTEXT_GUARD punch list friction
The gate requires a CONCLUDE PUNCH LIST with DOCS/CONTEXT/HONCHO fields in every submission. For trivial changes (1-line security fix), this is overhead with no value. The gate could default NOT_NEEDED and only require explicit confirmation when something was changed.

### SHA gate prose sensitivity
The regex in `sha_gate.py:56` matches ` failed ` with surrounding whitespace in ANY text, including the operator's own prose. If the VERIFY report says "...the first submission failed because..." the gate trips. This is the root of most BLOCK_HITL cycles — the fix is to scope the regex to command output lines only.

### Fast-mode CONCLUDE gap
RETRO returns LoopAction.DONE for fast-mode tasks, but subsequent CONCLUDE/REFRESH_CONTEXT submissions throw "task is in terminal state: DONE". This is technically correct but the error trace is confusing on first encounter.

## Pipeline improvements needed

| # | Problem | Proposed fix |
|---|---|---|
| 1 | CONTEXT_GUARD rejects generic summaries | Default DOCS/CONTEXT/HONCHO to NOT_NEEDED; require explicit confirmation only when changed |
| 2 | SHA gate triggers on "failed" in operator prose | Only match failure patterns in command output lines, not operator narrative text |
| 3 | Fast mode CONCLUDE/REFRESH_CONTEXT errors | Accept CONCLUDE and REFRESH_CONTEXT on DONE as no-op instead of throwing ValueError |
| 4 | Subagent timeouts leave partial uncommitted work | On timeout, check `git diff --stat` for partial changes and surface them instead of reporting empty failure |
| 5 | Subagent changes need automated verification gate | Auto-run `svelte-check` + `npm test` after subagent IMPLEMENT before accepting phase as complete |
| 6 | Multi-file subagent timeouts reduce net benefit | Replace multi-file delegation with single-file-per-task pipeline runs for <10 file changes |

## One recommendation for next session

**Replace multi-file subagent dispatches with single-file pipeline runs.** Each 5-minute subagent timeout costs more in verify + revert + redo than fixing the file directly. For changes touching 3+ files, run individual pipeline tasks per file instead of bundling.
